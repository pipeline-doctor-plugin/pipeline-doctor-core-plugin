Pipeline Doctor Configuration

Overview

Pipeline Doctor provides automated diagnostic analysis for Jenkins builds. It examines build logs, configuration, and artifacts to identify common issues and suggest solutions.

Configuration Options

Enable by Default

When enabled, all new jobs will automatically include Pipeline Doctor analysis. Existing jobs can still opt-in or opt-out individually through their configuration.

Auto-analyze Builds

When enabled, Pipeline Doctor will automatically analyze builds when they complete. If disabled, analysis must be triggered manually.

Maximum Results per Build

Limits the number of diagnostic results displayed per build to prevent UI overload. Results are prioritized by severity and confidence level.

Analysis Timeout

Maximum time allowed for diagnostic analysis. If analysis takes longer than this timeout, it will be terminated to prevent resource exhaustion.

Excluded Job Patterns

Regular expression patterns for job names that should be excluded from analysis. This is useful for excluding test jobs, temporary jobs, or jobs that don't benefit from analysis.

Examples:

  • .*-test$ - Exclude jobs ending with "-test"
  • temp/.* - Exclude all jobs in the "temp" folder
  • (nightly|weekly)-.* - Exclude jobs starting with "nightly-" or "weekly-"

Enable Detailed Logging

Enables verbose logging for diagnostic analysis. This is useful for debugging issues with the plugin but will increase log volume significantly.

Plugin Architecture

Pipeline Doctor uses an extensible plugin architecture:

  • Core Plugin: Provides base interfaces and UI
  • Diagnostic Providers: Individual plugins that analyze specific types of issues
  • Extension Points: Allow custom diagnostic logic

Performance Considerations

  • Analysis runs after build completion, not during build execution
  • Analysis is performed asynchronously to avoid blocking Jenkins
  • Results are cached to improve UI responsiveness
  • Resource usage scales with the number of enabled diagnostic providers

Troubleshooting

If you encounter issues:

  1. Enable detailed logging and check Jenkins logs
  2. Verify that diagnostic provider plugins are installed and enabled
  3. Check that jobs are not excluded by patterns
  4. Ensure adequate timeout settings for complex builds
